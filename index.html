Task 3: 10 Common Web Attacks
Name Web Attacks

Cross-Site Scripting

Source:

https://evalian.co.uk/wp-content/uploads/2022/03/XSS-attacks-what-is-cross-site-scripting.png

This attack relies on attackers injecting malicious code into vulnerable websites through compromised user interactions. The malicious code can perform actions such as stealing sensitive data like session tokens or cookies. One way this type of attack can be prevented by sanitising user input as it is received , to ensure that executable code is not injected or executed.

https://portswigger.net/web-security/cross-site-scripting

SQL Injection Attacks

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcQ5pnA5vbWEKmu-7c3D3n80kguJZgG95HDakA&s

SQL injection relies on exploiting input fields on a website. Instead of sending expected inputs, malicious SQL code is sent instead. Once this code is executed it can access sensitive data in the database if proper protections are not in place. This attack can also be preveneted through sanatizing user input.

https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

Phishing

Source:

href = "https://vipre.com/wp-content/uploads/2022/11/spear-phishing-1024x747.jpeg

Phising is a type of social engineering attack that imitate legitimate sources to trick users into revealing personal information. One common method is targetted emails appearing to be from reputable companies conveying a sense of urgency to the users, such as from: commmonweallthbank@email.com, subject: URGENT ACTION REQUIRED, message: Suspicious activity detected Please urgently provide your credit card informaiton to confirm your identity and secure your account. One way to prevent such attacks is increased awareness so victims know what to look out for such as urgent requests and slightly mispelled company names.

https://support.microsoft.com/en-us/windows/protect-yourself-from-phishing-0c7ea947-ba98-3bd9-7184-430e1f860a44#:~

Man in the Middle Attacks

Source:

https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/

This attack refers to a hacker placing themselves between a user and an application. Attackers can intercept legitimate communications to steal sensitive informaiton. Secure end to end ecryption is one protection against this, as even if an attacker intercepts web packets, they will not be able to decrypt them.

https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/

Dos/DDoS

Source:

https://cf-assets.www.cloudflare.com/slt3lc6tev37/1FIBEeoyzoa64lVGlWKaRV/3b878bb03df1729b48cd3f667cdfe3de/amplification_ddos_example.png

This stands for denail of service/distributed denial of service attacks. This is when a malicious actor uses machines to overwhelm a target For example, flooding a server with requests can cause it to crash or corrupt data.

https://www.fortinet.com/resources/cyberglossary/dos-vs-ddos

Clickjacking

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcSHLv6Y6SHfpPW24OAHOjcfjjar0MckoLU3eg&s

Cickjacking refers to deceiving a user into interacting with a malicious target site in ways they did not intend. One example of this is displaying a button on a web page with harmless text such as 'Click to load image'. Once clicked the user is actually interacting with hidden code from a malicious site. This is done by embedding the actual target site code inside an iframe, setting the iframes opacity to zero and placing it on top of the decoy button.

https://developer.mozilla.org/en-US/docs/Web/Security/Attacks/Clickjacking

Broken Authentication

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcS4b0oL6pVxGGsmMCORlLh_TZ4tQ-j4NkTT5A&s"

Systems are subject to broken authentication attacks when attackers are able to exploit weaknesses in authentication system to comporomise senstitive data such as passwords, tokens and user information. One leading cause of such an attack is poor code design and implementation of access controls. A website is also vulnerable to such an attack when credentials are not encrypted when they are sent. Improved design and implementation of access controls and identity management is a key fix for this attack.

https://www.contrastsecurity.com/glossary/broken-authentication#:~:text=Authentication%20is%20%E2%80%9Cbro// ken%E2%80%9D%20when%20attackers,of%20broken%20authentication%20is%20widespread

Click Fraud

Source:

https://multichannelmerchant.com/wp-content/uploads/2022/06/click-fraud-illustration-feature.jpg"

When bots and other malicious actors repeatedly click on adds this can lead to artificially inflated advertisement interactions. If not protected against such an attack, a company can spend millions on advertisments that real users are not engaging with. Companies can use automated click fraud protection to mitigate such attacks.

https://www.clixtell.com/click-fraud-protection/?gad_source=1&gclid=CjwKCAiAt4C-BhBcEiwA8Kp0C dB5bvf3c65yeLkooeIHIQMsUNvPKD_N2E5P3flTtYSV3pxmCRquqhoCgLUQAvD_BwE

HTTP Response Splitting

Source:

https://thecyphere.com/wp-content/uploads/2022/03/HTTP-Response-Splitting-Attack-768x292-1.webp

Web servers know how to respond to requests that come in an expected format. HTTP response splitting is a type of attack where HTTP responses can be wrapped in malicioius code. This may lead to the server executing the code and this may lead to malicious responses being sent out by the web server, Any incoming requests must be properly sanitised to protect against these types of attacks.

https://www.bitdefender.com/en-au/blog/hotforsecurity/understanding-iot-vulnerabilities-http-response-splitting

Fuzzing

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRsgxc2Zx2kRIPVcsLH0E3uy24R97grjcVPfw&s

This type of attack occurs when a hacker inputs large amounts of data into a website in the hopes that it will crash the site. Once the site has crashed the hackers uses software to analyse the site's weak spots, and then uses these weak spots to gain further access to the server. Updated security measures and installing the latests patches are mitigation techniques against fuzzing attacks.

https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself
