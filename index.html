Task 3: 10 Common Web Attacks
Name Web Attacks

Cross-Site Scripting

Source:

https://evalian.co.uk/wp-content/uploads/2022/03/XSS-attacks-what-is-cross-site-scripting.png

Cross site scripting is a type of attack that relies on injecting malicious code into websites. The attack works by exploiting vulnerabilities in the website through compromise user interactions. Once code is injected it can then execute in the browser and perform harmful actions, such as stealing senstivie information like cookies or session tokens.

One way to prevent such attacks is to properly validate and sanitise user input. Properly santitised input will ensure that no malicious code is executed.

https://portswigger.net/web-security/cross-site-scripting

SQL Injection Attacks

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcQ5pnA5vbWEKmu-7c3D3n80kguJZgG95HDakA&s

SQL injection refers to an attack that exploits a website's input fields. Malicious SQL code is injected into the vulnerabile input fields and, if executed, can potentially access the database. Once in the database an attacker perform actions including, but not limited to, manipulating data, stealing credentials and corrupting files.

Sanitising user input is also a protection against SQL injection attacks.

https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

Phishing

Source:

href = "https://vipre.com/wp-content/uploads/2022/11/spear-phishing-1024x747.jpeg

Phishing is a type of social engineering attack that imitates legitimate sources to trick users into revealing personal information. One common method is targeted emails appearing to be from reputable companies. Often such emails will convey a sense of urgency to the users. For examle an email may say:

From: commmonweallthbank@email.com

Subject: URGENT ACTION REQUIRED

Message: Suspicious activity detected. Please urgently provide your credit card informaiton to confirm your identity and secure your account.

One way to prevent such attacks is increased awareness and training. Since these attacks rely on tricking users training should focus on key indicators of phishing attacks, such as urgent request for personal informaiton, slightly mispelled company names and suspicious links in emails.

https://support.microsoft.com/en-us/windows/protect-yourself-from-phishing-0c7ea947-ba98-3bd9-7184-430e1f860a44#:~

Man in the Middle Attacks

Source:

https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/

This attack occurs when an attacker places themselves between the user and the application to intercept information from legitimate communications. If networks are not properly secured, the attacker can access sensitive information contained in the network packets. Some possible outcomes of such an attack include credential theft or redirecting the user to malicious sites.

End to end encryption is a key prevention for man in the middle attacks. Even if communication is intercepted, the attacker will not be able to decrypt packets to extract sensitive data.

https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/

Dos/DDoS

Source:

https://cf-assets.www.cloudflare.com/slt3lc6tev37/1FIBEeoyzoa64lVGlWKaRV/3b878bb03df1729b48cd3f667cdfe3de/amplification_ddos_example.png

Denial of Service (DoS) and Distributed Dential of Service (DDoS) attacks occur when a hacker overwhelms a target machine in an attempt to make the service unusable. This is ahcieved through flooding the target with requests. Such attacks can involve a network of compromised machines coordinating to crash the target service. This can result in the target crashing, slowing down to an unusable pace or even corrupting data.

https://www.fortinet.com/resources/cyberglossary/dos-vs-ddos

Clickjacking

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcSHLv6Y6SHfpPW24OAHOjcfjjar0MckoLU3eg&s

Clickjacking is a type of web attack that deceives the user into interacting with a malicious target site in ways they did not intend. One example of this is displaying a button on a web page with harmless text such as 'Click to load image'. Once clicked the user is actually interacting with a malicious website. This is done by embedding the malicious website inside an iframe, setting the iframe's opacity to zero and placing it on top of the decoy button.

https://developer.mozilla.org/en-US/docs/Web/Security/Attacks/Clickjacking

Broken Authentication

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcS4b0oL6pVxGGsmMCORlLh_TZ4tQ-j4NkTT5A&s"

Systems are subject to broken authentication attacks when attackers are able to exploit weaknesses in authentication system to compromise senstitive data such as passwords, tokens and user information. One leading cause of such an attack is poor design and weak implementation of access controls allowing a user to bypass security measures. A website is also vulnerable to such an attack when credentials are not properly encrypted in transit.

To prevent broken authentication secure coding practices, strong encryption, and robust access control mechanisms should be implemented.

https://www.contrastsecurity.com/glossary/broken-authentication#:~:text=Authentication%20is%20%E2%80%9Cbro// ken%E2%80%9D%20when%20attackers,of%20broken%20authentication%20is%20widespread

Click Fraud

Source:

https://multichannelmerchant.com/wp-content/uploads/2022/06/click-fraud-illustration-feature.jpg"

Click fraud occurs when bots and other malicious actors repeatedly click on adds, artificially inflating advertisement interactions. If not protected against such an attack, a company can spend millions on advertisments that real users are not engaging with.

To mitigate these attacks companies can use click fraud detection systems to identify and block suspicious activity.

https://www.clixtell.com/click-fraud-protection/?gad_source=1&gclid=CjwKCAiAt4C-BhBcEiwA8Kp0C dB5bvf3c65yeLkooeIHIQMsUNvPKD_N2E5P3flTtYSV3pxmCRquqhoCgLUQAvD_BwE

HTTP Response Splitting

Source:

https://thecyphere.com/wp-content/uploads/2022/03/HTTP-Response-Splitting-Attack-768x292-1.webp

HTTP Response splitting as a web attack where an attacker manipulates HTTP response headers through injecting malicious code into the response. When web servers receive HTTP requests in an unexpected format they may process the request incorrectly. This may cause the server to behave in a way that allows the attacker to alter the server's HTTP response. This may lead to issues such as compromised security and misleading content on the website.

Properly santitsing incoming requests and validating user input in a key prevention against such attacks.

https://www.bitdefender.com/en-au/blog/hotforsecurity/understanding-iot-vulnerabilities-http-response-splitting

Fuzzing

Source:

https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRsgxc2Zx2kRIPVcsLH0E3uy24R97grjcVPfw&s

Fuzzing is an attack where a hacker inputs large amounts of data into a website in the hopes that it will crash the site. The data is often random and malformed. Once the site has crashed the hackers uses software to analyse the site's weak spots, and then uses these weak spots to gain further access to the server.

Updated security measures and installing the latests patches are key mitigation techniques against fuzzing attacks.

https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself
